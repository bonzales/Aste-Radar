# docs/ — documentazione per l'umano (Diátaxis)

Organizzata secondo [Diátaxis](https://diataxis.fr/). Separa quattro bisogni
distinti; non mescolarli nello stesso documento:

- **`tutorials/`** — apprendimento guidato, orientato al fare. *"Il tuo primo
  giro di scansione."*
- **`how-to/`** — ricette per un obiettivo preciso. *"Come aggiungere un comune",
  "come cambiare una soglia."*
- **`reference/`** — descrizione asciutta e completa. Comandi, schema DB, formato
  JSON estratto.
- **`explanation/`** — comprensione e contesto. Perché questa architettura,
  lezioni dai progetti precedenti.

> Nota: la conoscenza di dominio per l'**agente** non sta qui ma in `research/`
> (CLAUDE.md §2). `docs/` è per l'umano; `research/` è la wiki LLM.

I singoli documenti arriveranno insieme alle fasi che descrivono (CLAUDE.md §4):
la documentazione segue la funzionalità, non la precede a vuoto.
